# 로직: 집중학습 큐레이션 (과목 · 유형 · 개념)

> 마이페이지에서 진입하는 3종 집중학습의 문제 선정 로직. 실제 코드 기준.

---

## 1. 공통 사항

| 항목 | 내용 |
|------|------|
| **최대 문항 수** | 50문항 (`WEAK_FOCUS_50_TARGET = 50`) |
| **모드 선택** | 학습모드 / 시험모드 선택 모달 → 선택 후 퀴즈 진입 |
| **과목 정렬** | 최종 결과는 `subject_number` 오름차순 정렬 |
| **부족 모달** | 0문항일 때만 alert, 1문항 이상이면 그대로 퀴즈 진행 |
| **진입 경로** | MyPage → 모드 선택 모달 → 3초 준비 오버레이 → `/quiz` |

---

## 2. 과목 강화 학습 (`fetchSubjectStrengthTraining50`)

`examService.ts` — 사용자가 선택한 과목 기반.

### 선정 로직

1. **subject_stats**에서 선택 과목의 `proficiency` 확인
2. 해당 과목 문제 풀에서:
   - **1순위**: 틀린 문제 (wrongIds) — 난이도 스캐폴딩 적용
   - **2순위**: 맞춘 문제 (correctIds) — 셔플 후 채움
3. 최대 50문항, 과목별 `subject_number` 순 정렬

### 난이도 스캐폴딩

| proficiency 대역 | 선호 난이도 |
|-------------------|-------------|
| Low (< 0.4) | 1, 2 우선 → 나머지로 채움 |
| Mid (0.4~0.7) | 2, 3 우선 → 나머지로 채움 |
| High (> 0.7) | 3, 4, 5 우선 → 나머지로 채움 |

---

## 3. 취약 유형 집중학습 (`fetchWeakTypeFocus50`)

`examService.ts` — 1회 이상 모의고사 응시 후 학습결과 기반.

### 선정 로직

1. **problem_type_stats**에서 proficiency 낮은 순으로 유형 정렬
2. 각 유형별 순회하며 50문항 채움:
   - **1순위**: 해당 유형 중 **틀린 문제** (wrongIds) — 셔플
   - **2순위**: 해당 유형 중 **맞춘 문제** (correctIds) — 셔플
   - **3순위**: 해당 유형 풀에서 **난이도 스캐폴딩** (`getProficiencyBand` → `getPreferredDifficultyLevels` → `pickByDifficultyThenFill`)
3. 1순위 유형에서 50개 미달 → 2순위 유형으로 이동, 동일 방식 반복
4. 최종 `subject_number` 순 정렬

### 유형 우선순위

```
weakTypeOrder = Object.entries(problemTypeStats)
  .filter(총시도 > 0)
  .sort(proficiency 오름차순)
  .map(키 추출)
```

### problem_type_stats 키 없는 경우

- `allPool`의 `problem_types[]`에서 유형 수집 → 순서대로 순회

## 4. 취약 개념 집중학습 (`fetchWeakConceptFocus50`)

`examService.ts` — 핵심 개념 이해도 기반.

### 선정 로직

1. **상위 3개 취약 개념 도출**:
   - `sub_core_id_stats` → `coreId`별 평균 proficiency 합산 → 낮은 순 3개
   - 없으면 `core_concept_stats` 상위 3개
2. 취약 개념 2~10개 풀 구성 (`conceptOrder` — proficiency 낮은 순)
3. 선정 순서:
   - **1순위**: 오답 중 **top3 개념** 문제 → 나머지 개념 오답
   - **2순위**: top3 개념 풀에서 **난이도 스캐폴딩** → 나머지 개념에서 스캐폴딩
   - **3순위**: 난이도 해제 후 50문항 채움
4. 최종 `subject_number` 순 정렬

### 핵심 개념 매칭

- 문제의 `hierarchy` 필드와 `core_concept_stats` 키 매칭
- `sub_core_id`가 있으면 세부 개념 수준까지 활용

## 5. 난이도 스캐폴딩 공통 함수

### `getProficiencyBand(proficiency01)`

| proficiency (0~1) | 대역 |
|--------------------|------|
| < 0.4 | `'low'` |
| 0.4 ~ 0.7 | `'mid'` |
| > 0.7 | `'high'` |

### `getPreferredDifficultyLevels(band)`

| 대역 | 선호 difficulty_level |
|------|----------------------|
| `low` | [1, 2] |
| `mid` | [2, 3] |
| `high` | [3, 4, 5] |

### `pickByDifficultyThenFill(pool, preferredLevels, seen, need)`

1. 선호 난이도에 해당하는 문제 우선 셔플 선택
2. 부족분은 남은 전체 문제에서 셔플 채움

---

## 6. 데이터 흐름

```
[진입] MyPage → 모드 선택 (학습/시험)
  → 3초 준비 오버레이
  → fetchSubjectStrengthTraining50 / fetchWeakTypeFocus50 / fetchWeakConceptFocus50
    → fetchExamResultAnswerSets(uid, certCode) — correctIds, wrongIds
    → fetchAllPoolQuestions(certCode) — 전체 문제 풀
    → fetchStatsDoc(uid, certCode) — 학습자 통계
  → 문제 선정 (우선순위별)
  → subject_number 순 정렬
  → /quiz 진입
```

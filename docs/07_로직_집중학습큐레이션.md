# 로직: 집중학습 큐레이션 (과목/유형/개념)

> 과목 강화, 취약 유형 집중, 취약 개념 집중 — 각 **50문항** 선정 로직.  
> 관련 파일: `src/services/examService.ts`, `src/App.tsx`, `src/pages/MyPage.tsx`.

---

## 1. 진입 흐름

1. **마이페이지**에서 버튼 클릭: "과목 강화" / "취약 유형 집중 학습" / "취약 개념 집중 학습".
2. **모드 선택 모달** (학습 모드 / 시험 모드) — `App.tsx`의 `pendingFocusTraining` 상태로 모달 표시.
3. **준비 오버레이** (딤 + 팝업): "N초 후 맞춤 문항을 준비합니다" → 5초 카운트다운 → "준비가 완료되었습니다" (약 1.5초 후 퀴즈 진입).
4. **50문항 큐레이션** 후 `/quiz`로 이동 (학습/시험 모드에 따라 테마만 구분).

- **상태 관리:** `App.tsx` — `focusPreparingType`, `focusPreparingPhase`, `focusPreparingCountdown`, `focusPreparingQuestions`, `focusPreparingCertId`, `focusPreparingMode`.  
- **카운트다운:** 5초. `focusPreparingPhase === 'ready'` 되면 문항 fetch 완료 후 약 1.5초 뒤 `navigate('/quiz')`.

---

## 2. 과목 강화 (fetchSubjectStrengthTraining50)

- **위치:** `examService.fetchSubjectStrengthTraining50(uid, certCode, subjectNumber)`.
- **대상:** 선택한 과목(`subject_number`)의 문항 **50개**.
- **조건:**  
  - 모의고사 1회 이상 응시 데이터 필요.  
  - 해당 과목 문항이 50개 이상 있어야 함.  
- **미충족 시:** `insufficient: true` 반환 → 앱에서 "아직 충분한 데이터가 쌓이지 않았어요. 조금 더 학습을 진행해주세요." 안내.
- **정렬:** 최종 문항 목록은 **subject_number 순** 정렬.
- **데이터 소스:** 인덱스 캐시 또는 Firestore 풀에서 해당 과목 필터 후 50개 선정.

---

## 3. 취약 유형 집중 (fetchWeakTypeFocus50)

- **위치:** `examService.fetchWeakTypeFocus50(uid, certCode)`.
- **통계 기준:** `users/{uid}/stats/{certCode}.problem_type_stats` — 유형별 proficiency(낮을수록 취약).
- **우선순위:**  
  1. 가장 취약 유형의 **틀린 문제** (최대 50).  
  2. 부족분: 가장 취약 유형의 **맞춘 문제**.  
  3. 그 다음: 해당 유형 **난이도 스캐폴딩** (proficiency 기반 선호 난이도).  
  4. 여전히 부족하면 **두 번째 취약 유형**부터 동일 순서로 반복.
- **0문항 시:** "선별된 문제가 없습니다. 모의고사를 1회 이상 응시한 뒤 이용해 주세요." (insufficient 모달 아님, 0건만 안내).
- **정렬:** 최종 목록 **subject_number 순**.

---

## 4. 취약 개념 집중 (fetchWeakConceptFocus50)

- **위치:** `examService.fetchWeakConceptFocus50(uid, certCode)`.
- **통계 기준:** `core_concept_stats` 또는 `sub_core_id_stats` — 개념별 proficiency(낮을수록 취약).
- **선정:** 틀린 문제 우선, 부족분은 난이도 스캐폴딩으로 채움.  
- **미충족 시:** 50개 미만이면 "아직 충분한 데이터가 쌓이지 않았어요" 안내 (insufficient).
- **정렬:** 최종 목록 **subject_number 순**.

---

## 5. 공통 상수·데이터

- **목표 문항 수:** 50 (`WEAK_FOCUS_50_TARGET` 등으로 관리되는 경우 있음).  
- **오답/정답 집합:** `fetchExamResultAnswerSets(uid, certCode)` — 최근 exam_results 기반 correctIds/wrongIds.  
- **풀:** `fetchAllPoolQuestions(certCode)` — BIGDATA는 인덱스 캐시 기반 스텁 가능, 필요 시 Firestore 풀.

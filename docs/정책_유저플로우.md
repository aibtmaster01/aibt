# 정책: 유저 플로우

> 사용자 여정별 화면 전환·데이터 흐름. 실제 코드 기준.

---

## 1. 라우트 구조

| 경로 | 페이지 | 접근 조건 |
|------|--------|-----------|
| `/` | Home | 비로그인 가능 |
| `/mypage` | 나의 학습 대시보드 | 로그인 필수 |
| `/exam-list` | 모의고사 회차 목록 | certId 기준 |
| `/quiz` | 퀴즈 진행 | 회차·문제 데이터 필요 |
| `/result` | 결과 | 퀴즈 완료 후 |
| `/checkout` | 결제 | certId 기준 |
| `/account-settings` | 계정 설정 | 로그인 필수 |
| `/admin` | 관리자 | isAdmin 필수 |

## 2. 시나리오별 플로우

### A. 비로그인(게스트) → 모의고사 → 로그인

```
홈(/) → "무료 진단 시작" 클릭
  → 모의고사 목록(/exam-list) — Round 1만 노출
    → 퀴즈(/quiz) — 20문항 제한
      → 20번 제출 시 로그인 유도 모달 → 3초 후 /login
        → 로그인 성공
          → pendingGuestContinue 있으면 → 퀴즈 복귀(이어풀기)
          → pendingCheckoutCertId 있으면 → /checkout
          → 그 외 → /mypage
```

### B. 무료회원 → 모의고사 → 결제

```
홈 또는 마이페이지 → 모의고사 목록
  → Round 1~2 풀기 가능
  → Round 3 이상 클릭 → "열공모드로 합격권에 진입하세요" → /checkout
```

### C. 유료회원(열공모드) 전체 플로우

```
마이페이지(/mypage)
  ├── D-Day, 성적 추이, 약점 분석 대시보드 확인
  ├── "모의고사 시작하기" → 모의고사 목록
  │   ├── Round 1~3: 고정형 문제
  │   └── Round 6~20: 약점 공략 맞춤형 (Round 3 완료 후 순차 해제)
  ├── 과목 강화 학습 → 모드 선택(학습/시험) → 50문항 퀴즈
  ├── 유형 집중 학습 → 모드 선택 → 50문항 퀴즈
  └── 개념 집중 학습 → 모드 선택 → 50문항 퀴즈

퀴즈(/quiz) → 제출
  → 결과(/result) — 과목별 점수, 합격 여부, 오답 해설
  → 채점: exam_results 저장 + stats 3차원 갱신 + Elo 반영
  → "다음 회차" CTA → 다음 모의고사 자동 큐레이션
```

### D. 결제 → 이용권 활성화

```
결제(/checkout) → 회차 선택 (2개 옵션 시 모달, 1개 시 자동)
  → 결제 완료 → Pass(status: pending) 저장
  → /mypage 이동

시험당일 12:00 KST 이후 앱 진입
  → pending → active 자동 전환
  → "열공모드 활성화" 팝업
  → 전 회차 + 맞춤형 + 약점 공략 해제
```

## 3. 전환 유도 포인트

| 시점 | 대상 | 유도 |
|------|------|------|
| 20문항 완료 | 게스트 | "결과 저장 후 이어서 풀어보세요" → 로그인 |
| Round 3 잠금 클릭 | 무료회원 | "열공모드로 합격권에 진입하세요" → 결제 |
| 약점·AI 해설 마스킹 | 무료회원 | "이용권 구매 후 확인하기" → 결제 |
| 이용권 활성화 | 결제회원 | "열공모드 활성화" 팝업 → 인지도 상승 |
| D+1 시험 후 | 유료회원 | 합격 환급 / 불합격 재도전 할인 → 재전환 |
| 이용권 만료 | Expired | "오답 데이터 보관 중" → 재수강 CTA |

## 4. 데이터 관점 플로우

```
[인증] Firebase Auth + Firestore users/{uid}
  → subscriptions, paidCertIds, passesByCert 기반 등급 판별

[문제 선정]
  Round 1~3: static_exams/Round_{N} → 고정 문제 세트
  Round 6~20: aiRoundCurationService → Zone A(오답)/B(미풀이)/Fallback 배분

[채점] submitQuizResult
  → exam_results 저장
  → stats 3차원 갱신 (core_concept/problem_type/subject)
  → proficiency(Elo) 순차 갱신
  → elo_rating_by_cert 종합 갱신

[대시보드] statsService
  → 성적 추이, 레이더, 과목 밸런스, 약점 카드
  → proficiency → eloToPercent(0~100%)
```

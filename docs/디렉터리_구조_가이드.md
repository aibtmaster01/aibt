## 전체 구조 한눈에 보기

```
aibt_cursor/
├── backend/          ← Python 스크립트 (DB 시드 등)
├── docs/             ← 문서 (지금 보고 있는 것처럼)
├── src/              ← 프론트엔드 전체 (여기가 메인!)
│   ├── components/   ← 재사용 UI 컴포넌트
│   ├── contexts/     ← 전역 상태 (로그인 정보 등)
│   ├── pages/        ← 화면별 페이지
│   ├── services/     ← API·Firebase 호출 등 비즈니스 로직
│   └── utils/        ← 유틸 함수들
├── index.html
├── package.json
├── vite.config.ts    ← Vite 설정
├── tailwind.config.js
├── firebase.json     ← Firebase 설정
└── ...
```

## 2. `backend/` - 백엔드

Python 스크립트들입니다. **문제 생성 파이프라인**(AI-Gen Framework)과 Firebase 시드 등에 사용됩니다.

| 경로 | 하는 일 |
|------|---------|
| `Contents/Default/` | **범용** 문제 생성: Blueprint Extractor, Generator, Auditor (시험 무관 적용) |
| `Contents/SQLD_raw/` | **SQLD** 첫 타자: DNA 추출 → 초안 생성 → Gap 분석 → 3-Way Auditor → 메타데이터 주입 → 퍼널 배정 |
| `Contents/BIGDATA_raw/` | 빅데이터분석기사 적용 (진행 중) |
| `seed_users.py` 등 | Firebase Admin, 시드 데이터 |
| `requirements.txt` | Python 패키지 목록 |

→ 상세 기술·특허 청구항: `docs/AIBT_문제생성로직.md` 참고.

## 4. `src/` - 프론트엔드 (핵심)

React + TypeScript + Vite로 만든 화면·로직이 전부 여기 있습니다.

### 4.1. 최상위 파일

| 파일 | 하는 일 |
|------|---------|
| `index.tsx` | React 앱을 `#root`에 마운트하는 진입점 |
| `index.css` | 전역 스타일, Tailwind `@tailwind` 설정 |
| `App.tsx` | **라우팅·전체 흐름의 중심** – 페이지 전환, 핸들러, 상태 |
| `firebase.ts` | Firebase 초기화 (Auth, Firestore 연결) |
| `constants.ts` | 자격증 목록, 시험 회차, 테스트 유저 등 상수 |

## 6. 페이지 흐름 간단 요약

```
메인(/) → [무료 진단] → 자격증 선택 → Quiz → Result
         [로그인]   → Login → 마이페이지(/mypage)
                              → D-Day·성적추이·약점(정밀진단) 확인
                              → ExamList → 1~3회 고정 / 4회 맞춤형(20문항) / 5회 실전·약점(80문항)
                              → Quiz → Result → stats·Elo 갱신
                              → Checkout(결제)
```

- **4회차**: 항상 동일 큐레이션(Zone A/B, 과목 배분). UI에서만 "실전 대비" vs "약점 강화" 추천 문구·버튼 순서 변경.
- **5회차**: 실전 대비형(REAL_EXAM) 또는 약점 강화형(WEAKNESS_ATTACK)에 따라 문항 구성이 달라짐. 상세는 `AIBT_마스터_리포트_예창패_요청사항.md` 참고.

# AIbT 프로젝트 디렉터리 구조 가이드

> 파일이 뭐 하는지, 어디를 봐야 할지 헷갈릴 때 참고.

---

## 전체 구조 한눈에 보기

```
aibt_cursor/
├── backend/          ← Python 스크립트 (문제 생성, DB 시드)
├── docs/             ← 문서 (정책, 로직, 가이드)
├── src/              ← 프론트엔드 전체 (여기가 메인!)
│   ├── components/   ← 재사용 UI 컴포넌트
│   ├── contexts/     ← 전역 상태 (로그인 정보 등)
│   ├── data/         ← 정적 데이터 (코어컨셉, 목업)
│   ├── pages/        ← 화면별 페이지
│   ├── services/     ← API·Firebase 호출 등 비즈니스 로직
│   └── utils/        ← 유틸 함수들
├── index.html
├── package.json
├── vite.config.ts
├── tailwind.config.js
├── firebase.json
└── ...
```

## 2. `backend/` - 백엔드

Python 스크립트. **문제 생성 파이프라인**(AI-Gen Framework)과 Firebase 시드.

| 경로 | 하는 일 |
|------|---------|
| `Contents/Default/` | 범용 문제 생성 (Blueprint Extractor, Generator, Auditor) |
| `Contents/SQLD_raw/` | SQLD 문제 생성 파이프라인 |
| `Contents/BIGDATA_raw/` | 빅데이터분석기사 적용 (진행 중) |
| `seed_users.py` 등 | Firebase Admin, 시드 데이터 |
| `requirements.txt` | Python 패키지 목록 |

→ 상세: `docs/08_로직_문제생성로직.md` 참고.

## 4. `src/` - 프론트엔드 (핵심)

React + TypeScript + Vite.

### 4.1. 최상위 파일

| 파일 | 하는 일 |
|------|---------|
| `index.tsx` | React 앱을 `#root`에 마운트하는 진입점 |
| `index.css` | 전역 스타일, Tailwind `@tailwind` 설정 |
| `App.tsx` | **라우팅·전체 흐름의 중심** — 페이지 전환, 핸들러, 상태 |
| `firebase.ts` | Firebase 초기화 (Auth, Firestore 연결) |
| `constants.ts` | 자격증 목록, 시험 회차, 시험 일정 등 상수 |
| `types.ts` | TypeScript 타입 (`User`, `Question`, `Certification` 등) |

### 4.2. `src/pages/` - 화면별 페이지

| 파일 | 경로 | 하는 일 |
|------|------|---------|
| `Home.tsx` | `/` | 메인 랜딩, CTA, 서비스 소개 |
| `MyPage.tsx` | `/mypage` | 마이페이지 (학습 현황, D-Day, 추천 등) |
| `ExamList.tsx` | `/exam-list` | 시험 목록 (회차 선택) |
| `Quiz.tsx` | `/quiz` | 문제 풀이 화면 |
| `Result.tsx` | `/result` | 시험 결과 화면 |
| `Checkout.tsx` | `/checkout` | 결제 화면 (모달) |
| `AccountSettings.tsx` | `/account-settings` | 계정 설정 |
| `Admin.tsx` | `/admin` | 관리자 전용 |
| `AdminCerts.tsx` | `/admin/certs` | 관리자 자격증 관리 |
| `AdminQuestions.tsx` | `/admin/questions` | 관리자 문제 관리 |

### 4.3. `src/components/` - 공통 컴포넌트

| 파일 | 하는 일 |

### 4.4. `src/contexts/` - 전역 상태

| 파일 | 하는 일 |
|------|---------|
| `AuthContext.tsx` | 로그인 상태 (`user`, `loading`), `login`/`logout`/`updateUser` |

### 4.5. `src/services/` - 비즈니스 로직

| 파일 | 하는 일 |
|------|---------|
| `authService.ts` | 로그인·회원가입·Firestore 유저 문서 생성 |

| `statsServiceWithCache.ts` | 마이페이지 캐시 래퍼 |
| `adminService.ts` | 관리자 회원/멤버십/통계 |
| `adminQuestionService.ts` | 관리자 문제 CRUD |
| `errorLogService.ts` | 클라이언트 에러 Firestore 로깅 |

### 4.7. `src/data/` - 정적 데이터

| 파일 | 하는 일 |

## 5. "이거 하려면 어디를 봐야 하지?" 빠른 참고

| 하고 싶은 것 | 열어볼 파일 |
|--------------|-------------|
| 화면 구성/라우팅 바꾸기 | `App.tsx` |
| 로그인 상태 쓰거나 바꾸기 | `AuthContext.tsx` |
| 메인 화면 수정 | `Home.tsx` |
| 마이페이지 수정 | `MyPage.tsx` |
| 4회차 추천·모달 문구 | `ExamList.tsx` |
| 맞춤형 문항 구성 로직 | `aiRoundCurationService.ts`, `examService.ts` |
| 정밀진단(통계·Elo) | `gradingService.ts`, `statsService.ts` |
| 문제 풀이 화면 수정 | `Quiz.tsx` |
| 결과 화면 수정 | `Result.tsx` |
| Firestore 데이터 | `authService`, `gradingService`, `examService` 등 |
| 자격증·시험 회차 추가 | `constants.ts` |
| 타입 정의 보기/추가 | `types.ts` |
| 전역 스타일/색상 | `index.css`, `tailwind.config.js` |

## 6. 페이지 흐름

```
메인(/) → [무료 진단] → 자격증 선택 → Quiz → Result
         [로그인 모달] → 마이페이지(/mypage)
                         → D-Day·성적추이·약점(정밀진단) 확인
                         → ExamList → 1~3회 고정 / 4·5회 고난도 / 6+회 약점 공략
                         → Quiz → Result → stats·Elo 갱신
                         → Checkout(결제 모달)
```

---

## 7. 참고하면 좋은 순서

1. `App.tsx` — 전체 흐름 파악
2. `types.ts` — 데이터 구조 파악
3. `AuthContext.tsx` — 로그인 상태 흐름
4. `Home.tsx` → `MyPage.tsx` — 메인 → 마이페이지
5. `Quiz.tsx` → `Result.tsx` — 문제 풀이 → 결과
6. `services/` — 실제 데이터 읽기/쓰기 로직

## 한눈에 보는 시스템 구조

```
[문제 생성]  backend/Contents/     Python 파이프라인 (LLM → 검수 → Firestore)
     ↓
[문제 저장]  Firestore              certifications/{cert}/question_pools/.../questions/{q_id}
     ↓
[큐레이션]  src/services/           Zone 기반 적응형 문항 선정 + Elo 기반 난이도 스캐폴딩
     ↓
[채점·진단] src/services/           3차원 통계(개념·유형·과목) + Elo proficiency 갱신
     ↓
[대시보드]  src/pages/MyPage.tsx    레이더/과목 밸런스, 약점 분석, 예측 합격률
```

## 핵심 알고리즘 상세

### A. Zone 기반 적응형 큐레이션 (`aiRoundCurationService.ts`)

```
입력: 유저의 exam_results(정오답 이력) + stats(proficiency) + 문제 풀 인덱스
     ↓
1. 약점 Zone (40%): proficiency 낮은 sub_core_id 문제 우선
2. 강점 Zone (30%): proficiency 높은 영역에서 도전 문항
3. 랜덤 Zone (30%): 미응시 문제 랜덤 배분
     ↓
과목별 20문항 × 4과목 = 80문항 (1→2→3→4 과목 순서 고정)
```

### B. Elo Proficiency 시스템 (`gradingService.ts`)

```
Expected = 1 / (1 + 10^((1200 - Proficiency) / 400))
NewP = Old + 32 × (Outcome - Expected)

Lucky-Guess 보정: 정답 + 헷갈림 체크 → delta × 0.2
범위: 100 ~ 2500, 기본값 1200
```

- 개념별/유형별/과목별 독립 proficiency 유지
- 종합 Elo: 회차 전체 정답률 기반 갱신

### C. 집중학습 난이도 스캐폴딩 (`examService.ts`)

```
proficiency → band (low/mid/high)
  low  (< 0.4): 난이도 1, 2 우선
  mid  (0.4~0.7): 난이도 2, 3 우선
  high (> 0.7): 난이도 3, 4, 5 우선

각 유형 내 우선순위:
  1순위: 틀린 문제 → 2순위: 맞춘 문제 → 3순위: 난이도 스캐폴딩
```

### D. 3차원 통계 집계 (`gradingService.ts`)

```
제출 시 문항별로 4가지 통계 갱신:
  core_concept_stats  — 핵심 개념별 (correct, total, proficiency)
  problem_type_stats  — 문제 유형별 (1문항 → 여러 유형에 기여)
  subject_stats       — 과목별
  sub_core_id_stats   — 세부 개념별

confused_qids — 헷갈림 체크 문항 ID (최근 100개)
```

## Firestore 데이터 구조

```
certifications/{certCode}/
  certification_info/config        ← 과목·배점·합격 기준
  question_pools/{poolId}/
    questions/{q_id}               ← 문제 본문·선지·해설·메타데이터
  static_exams/Round_{N}           ← 고정 시험 장부 (Round 1~3)
  public/index                     ← 문제 인덱스 (캐싱용)

users/{uid}/
  elo_rating_by_cert: { c1: 1230 } ← 종합 Elo
  exam_results/{examId}            ← 시험 결과 (점수, 합격, 답안)
  stats/{certCode}                 ← 3차원 통계 + proficiency
  user_rounds/{roundNum}           ← 회차별 고정 문제 세트 (박제)
```

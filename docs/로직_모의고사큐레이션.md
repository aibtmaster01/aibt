# 로직: 모의고사 큐레이션

> 고정형 모의고사(Round 1~3) 및 약점 공략 맞춤형(Round 6~20) 문제 선정 로직. 실제 코드 기준.

---

## 1. 회차별 문제 선정 구분

| 회차 | 문항 수 | 유형 | 선정 방식 |
|------|---------|------|-----------|
| **Round 1~3** | 80 | 고정형 | `static_exams/Round_{N}`의 `question_refs` (동일 문제 고정) |
| **Round 6~20** | 80 | 약점 공략 맞춤형 | Zone A/B/Fallback + 과목별 20문항 배분 |

---

## 2. 회차 노출 조건 (ExamList.tsx)

| 조건 | Round 1~3 | Round 6~20 |
|------|-----------|------------|
| 항상 노출 | O (순차 해제) | - |
| Round 3 완료 | - | O (순차 해제) |
| 이전 회차 완료 | 필수 | 필수 |

- 최대 **20회차**까지 동적 생성 (`ExamList.tsx` — `MAX_CURATION_ROUND = 20`)

---

## 3. UserRound 박제 시스템

유저가 특정 회차에 최초 진입하면 **문제 세트를 고정(박제)**하여 재응시 시 동일 문제 유지.

| Firestore 경로 | 필드 |
|----------------|------|
| `users/{uid}/user_rounds/{roundNum}` | `roundNum`, `sourceRounds`, `questionIds[]`, `createdAt` |

**동작**:
1. 모의고사 진입 → `user_rounds/{roundNum}` 조회
2. **있으면**: 저장된 `questionIds`로 즉시 로드 (재응시 시 동일 문제)
3. **없으면**: Static vs 맞춤형 판단 → 문제 생성 → Firestore 트랜잭션으로 UserRound 저장

---

## 4. Zone 기반 문항 배분 (Round 6~20)

`aiRoundCurationService.ts` — `selectQuestionIdsBy3ZonesPerSubject`

### 4.1 과목별 20문항 배분

80문항 = 4과목 × 20문항. 과목 1 → 2 → 3 → 4 순서로 출력.

### 4.2 Zone 구분

| 우선순위 | Zone | 설명 | 선정 기준 |
|----------|------|------|-----------|
| 1 | **Zone A (복습)** | 과거 틀린 문제 | `exam_results.answers`에서 `isCorrect === false` |
| 2 | **Zone B (도전)** | 안 푼 새로운 문제 | 풀에서 correctIds·wrongIds 모두 미포함 |
| 3 | **Fallback** | 맞춘 문제 재활용 | 가장 오래전에 푼 문제 또는 trap_score 높은 순 |

### 4.3 약점 유형 우선 선정

- `problem_type_stats`에서 **proficiency 낮은 유형**의 문제를 Zone A/B에서 우선 선택
- 약점 우선순위 공식:

```
Priority = (100 - Proficiency) × 0.5 + DaysSince × 0.3 + MisconceptionCount × 5 × 0.2
```

---

## 5. Lucky-Guess Elo 보정

채점 시 **정답 + 헷갈림 체크** → "운으로 맞춘 것"으로 간주:

- Elo 상승 폭에 **0.2(20%)**만 반영
- `gradingService.nextProficiency(old, outcome, isConfused)` 처리

---

## 6. 데이터 흐름 요약

```
[진입] getQuestionsForRound(certId, roundNum, user)
  → user_rounds/{round} 조회
  → 있으면: questionIds로 즉시 로드
  → 없으면: Static vs 맞춤형 판단
    → Round 1~3: static_exams/Round_{n}
    → Round 6+: aiRoundCurationService (Zone A → B → Fallback, 과목별 배분)
  → UserRound 저장 (트랜잭션)
  → maskQuestionData(등급별 AI해설 마스킹) → 반환

[제출] submitQuizResult
  → exam_results 저장
  → stats 3차원 갱신 → proficiency(Elo) 갱신
```

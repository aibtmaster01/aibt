# 마이페이지 (나의 학습 대시보드) 디자인 기능명세서

> 디자이너 전달용 상세 문서. 리디자인 시 참고할 화면 구조, 컴포넌트 의도, 기능, 데이터 흐름을 정리했습니다.

---

## 1. 페이지 개요

| 항목 | 내용 |
|------|------|
| **페이지명** | 나의 학습 대시보드 (MyPage) |
| **경로** | `/mypage` |
| **접근 조건** | 로그인 필수 (비로그인 시 메뉴에서 숨김) |
| **목적** | 사용자의 학습 진도, 약점 분석, D-Day, 모의고사 결과를 한눈에 보여주고 학습 동선(모의고사 입장, 약점 다시풀기)으로 연결 |

---

## 2. 사용자 상태별 화면 분기

| 상태 | 조건 | 화면 |
|------|------|------|
| **Empty State** | `subscriptions.length === 0` | 자격증 추가 유도 카드 3개 (BIGDATA, SQLD, ADsP) |
| **Regular** | 구독 있음, 수강 미만료 | 전체 대시보드 |
| **Expired** | `expiredCertIds`에 현재 자격증 포함 | 전체 대시보드 + 회색 톤 + 클릭 차단 + 만료 배너/재수강 CTA |

---

## 3. 화면 구성 (Regular State 기준, 상→하)

### 3.1. D+1 설문 배너 (조건부)
- **노출 조건**: `daysLeft < 0` (시험일 18시 KST 경과 후)
- **의도**: 시험 후 합격/불합격 피드백 수집, 불합격 시 50% 할인 유도
- **구성**:
  - 문구: "이번 시험 결과는 어떠셨나요?"
  - 버튼 2개: "합격했어요 🎉" (초록), "아쉬워요 😢 다음에 다시 도전할래요" (회색)
- **설계 참고**: 한 번 응답 후 재노출 방지 플래그(`hasAnsweredPostSurvey`) 사용

---

### 3.2. 만료 배너 (조건부)
- **노출 조건**: `expiredCertIds`에 현재 선택된 자격증 포함
- **의도**: 수강 만료 안내 + 재수강(50% OFF) CTA
- **구성**: Lock 아이콘, "수강 기간이 종료되었습니다.", "재수강 신청하기 (50% OFF)" 버튼

---

### 3.3. 자격증 탭 + 시즌 드롭다운
- **자격증 탭**: `user.subscriptions` 기준 탭 (빅데이터분석기사, SQLD 등)
- **선택 탭**: `activeCertId` — 이후 D-Day, 차트, 약점 등 모두 이 자격증 기준
- **자격증 추가**: "+ 자격증 추가" 버튼 → 모달 오픈
- **시즌 드롭다운**: "최신 회차" / "2026년 1회 (4월 8일)" 등 — `EXAM_SCHEDULES[activeCert.code]` (선택 사항, 현재는 로직만 준비)

---

### 3.4. D-Day 영역 (Hero)
- **의도**: 시험일까지/이후 D-Day 강조, 학습 동기 부여
- **구성**:
  - 자격증명 (예: 빅데이터분석기사)
  - 대형 숫자: `D-35` / `D+1` (양수=D-N, 음수=D+N)
  - 다음 시험일 라벨 (예: "2026년 1회 (4월 8일)")
  - 장식: Calendar 아이콘 (우하단, 저투명도)
- **데이터**: `getDaysLeft()`, `getNearestExamDate()` — 한국시간 18시 컷오프 적용

---

### 3.5. 오늘의 추천 학습 CTA
- **의도**: 바로 모의고사 목록으로 진입 유도
- **노출 조건**: `!isExpired`
- **구성**: "오늘의 추천 학습 시작하기", 부가 문구, ChevronRight 아이콘
- **클릭**: 모의고사 목록(`/exam-list`)으로 이동

---

### 3.6. 성적 향상 추이 (Line Chart)
- **의도**: 회차별 점수 추이로 실력 변화 확인
- **데이터**: 회차명(1회~7회), 점수 (0~100)
- **인터랙션**:
  - X축 라벨/차트 점 클릭 → 해당 회차 팝업
  - 팝업: 날짜, 점수, 합격/불합격 뱃지, "재응시", "오답 보기" 버튼 (현재 alert 처리)
- **설계 참고**: 가로 스크롤 가능, 최소 너비 700px

---

### 3.7. 집중 공략 필요 (Weakness Card)
- **의도**: **정밀진단** 기반 취약 유형 표시 + 약점 문제 다시풀기 진입
- **데이터 출처**: `users/{uid}/stats/{certCode}`의 hierarchy_stats·tag_stats. `statsService.fetchWeaknessFromStats`로 숙련도(proficiency→%)·정답률 낮은 순 항목 조회. (5회차 약점 큐레이션·calculatePriority와 동일 stats 사용)
- **구성**:
  - 제목 + AlertTriangle 아이콘
  - 취약 유형 리스트 (예: 통계적 추론 42%, 딥러닝 기초 48%)
  - 버튼: "약점 문제 다시풀기" (프리미엄만 활성)
- **프리미엄 여부**:
  - `isPremiumCert` = true: 정상 표시, 버튼 클릭 → 약점 문제 다시풀기(퀴즈)
  - `isPremiumCert` = false: dim + Lock 오버레이, 클릭 시 결제(`/checkout`) 이동

---

### 3.8. 예측 합격률 (Stats Card)
- **의도**: 최근 모의고사 기반 예측 합격률로 학습 방향 제시
- **구성**: 퍼센트 숫자, 진행 바, "합격 안정권까지 N점 남았습니다" 문구
- **프리미엄 여부**: 집중 공략 필요와 동일 (dim + Lock 오버레이)

---

### 3.9. 유형별 밸런스 (Radar Chart)
- **의도**: **정밀진단** — 문제 유형(problem_type)별 숙련도 시각화 (Elo proficiency → eloToPercent 0~100%)
- **데이터**: `users/{uid}/stats/{certCode}.problem_type_stats` (또는 hierarchy_stats) → `statsService.fetchRadarData()` (subject, A, fullMark)
- **상태**:
  - 로딩: "로딩 중..."
  - 데이터 부족: "아직 데이터가 부족합니다. 모의고사를 1회 이상 완료하면 AI 밸런스 분석이 활성화됩니다."
  - 데이터 있음: RadarChart 렌더
- **프리미엄 여부**: 집중 공략 필요와 동일

---

### 3.10. 과목별 밸런스 (Gauge Chart)
- **의도**: **정밀진단** — 과목(subject)별 숙련도·점수 시각화
- **데이터**: `users/{uid}/stats/{certCode}.subject_stats` → `statsService.fetchSubjectStatsRadar()` (과목별 proficiency 또는 correct/total → 점수). 자격증별 구조에 따라 MOCK fallback 가능
- **색상**: 80+ 초록, 60+ 노랑, 60 미만 빨강
- **프리미엄 여부**: 집중 공략 필요와 동일
- **레이아웃**: 유형별 밸런스와 동일 높이(`min-h-[360px]`)

---

### 3.11. 만료 유저용 데이터 보관 안내
- **노출 조건**: `isExpired`
- **의도**: 오답 데이터 보관 사실로 재수강 유도
- **구성**: 아이콘, "회원님의 오답 데이터 N개가 보관되어 있습니다", 부가 문구

## 4. 모달

### 4.1. 자격증 추가 모달
- **트리거**: "+ 자격증 추가" 클릭
- **의도**: 새 자격증 선택 후 구독 추가 (준비 중 제외)
- **구성**:
  - 제목: "새로운 도전 시작하기"
  - 자격증 카드 3개 (BIGDATA, SQLD, ADsP)
  - 이미 구독 중: disabled, "이미 구독중" 뱃지
  - 미구독: 클릭 시 해당 자격증 추가 시도 (준비 중이면 alert 후 닫기만)

---

### 4.2. D+1 합격 설문 모달
- **트리거**: D+1 배너에서 "합격했어요 🎉" 클릭
- **의도**: 합격 후기/명예의 전당 유도
- **구성**: 축하 문구, "후기 남기기" 버튼, "나중에 할게요" 버튼

---

### 4.3. D+1 불합격 50% 할인 모달
- **트리거**: D+1 배너에서 "아쉬워요 😢" 클릭
- **의도**: 50% 할인으로 재수강 유도
- **구성**: "50% 할인" 뱃지, "50% 할인 적용하여 결제하기" 버튼, "나중에 할게요" 버튼

---

## 5. Empty State (구독 없음)

- **제목**: "나의 학습 대시보드"
- **부제**: "아직 학습 기록이 없습니다. 자격증을 추가하고 시작해보세요."
- **카드 3개**: BIGDATA, SQLD, ADsP — 아이콘, 이름, 설명, "추가하기" CTA
- **클릭**: 해당 자격증 추가 시도 (준비 중이면 alert만)

## 6. 데이터/비즈니스 요약

| 항목 | 설명 |
|------|------|
| **isPremiumCert** | `user.isPremium` 또는 `user.paidCertIds`에 해당 자격증 포함 여부 |
| **isExpired** | `user.expiredCertIds`에 해당 자격증 포함 여부 |
| **daysLeft** | 다음 시험일 기준 D-N(양수) 또는 D+N(음수), 18시 KST 컷오프 |
| **정밀진단 데이터** | `users/{uid}/stats/{certCode}` — hierarchy_stats, problem_type_stats, subject_stats, tag_stats, confused_qids. 제출 시 gradingService가 3차원 통계·proficiency(Elo)·confused_qids 갱신. 종합 Elo는 `users/{uid}.elo_rating_by_cert`. |
| **성적 추이** | `statsService.fetchTrendData` — `users/{uid}/exam_results` 조회 후 포맷 변환 |
| **유형별·과목별 밸런스** | `statsService.fetchRadarData`, `fetchSubjectStatsRadar` — stats 내 proficiency → eloToPercent(0~100%)로 레이더/게이지용 |
| **집중 공략(약점 카드)** | `statsService.fetchWeaknessFromStats` — hierarchy_stats·tag_stats 하위 항목(정답률/숙련도 낮은 순). 5회차 큐레이션·약점 다시풀기와 동일 stats 기반 |

## 8. 반응형/레이아웃

- **최대 너비**: `max-w-5xl` (1024px)
- **패딩**: `px-5 py-10`
- **그리드**:
  - 집중 공략 + 예측 합격률: 1열(모바일) / 2열(데스크톱)
  - 유형별 + 과목별 밸런스: 1열(모바일) / 2열(대형)
- **카드**: `rounded-3xl`, `border-2 border-slate-300`

## 10. 개선 제안 (리디자인 시 고려)

1. **D-Day Hero**: 시험일 정보를 더 명확히, 캘린더 아이콘 활용
2. **차트 영역**: 카드 간 시각적 위계 정리
3. **프리미엄 Lock 오버레이**: 일관된 디자인, CTA 문구 표준화
4. **모달**: 통일된 레이아웃, 버튼 계층 구조
5. **Empty State**: 자격증 카드 비주얼 강화

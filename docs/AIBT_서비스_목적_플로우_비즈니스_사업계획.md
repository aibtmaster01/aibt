# AIbT 서비스 목적·플로우·비즈니스 모델·사업계획

> 예비창업패키지 및 사업계획서 작성 시 참고할 **서비스 목적, 사용자 플로우, 비즈니스 모델, 사업계획 관련** 정리 문서입니다.  
> 기술 스펙·코드 기준 상세는 `AIBT_마스터_리포트_예창패_요청사항.md`를 참고하세요.

## 1. 서비스 목적

### 1.1 핵심 목적

| 목적 | 설명 |
|------|------|
| **AI 기술력(MAV)에 대한 신뢰 부여** | “AI가 문제를 만든다”가 아니라 **“최신 기출 데이터를 학습한 AI 에이전트가 검수한다”**는 메시지로 데이터 근거(기출)를 명시하여 신뢰를 구축합니다. |
| **자격증 커리어 로드맵 제시** | ADsP(기초) → SQLD(중급) → 빅데이터분석기사(심화) 순서의 **성장 경로 시각화**로 사용자 LTV(생애 가치)를 높입니다. |
| **진단 테스트 전환 극대화** | 무료 진단(게스트 20문항, 무료회원 2회차)으로 진입을 낮추고, 로그인·결제(열공모드)로 전환합니다. |

### 1.2 서비스 한 줄 요약

- **데이터 자격증(빅데이터분석기사, SQLD, ADsP 등) 수험생을 위한 AI 기반 모의고사·약점 분석 서비스.**  
  기출 풀 기반 맞춤형 회차(Round 4+), 약점 공략(Round 5), 3차원 통계(개념·유형·과목·태그)로 **데이터 기반 합격 가이드**를 제공합니다.

### 1.3 법적·표현 가이드라인

- **“100% 합격 보장”** 사용 금지. 대신 **“합격 확률 최적화(Optimization)”**, **“데이터 기반 합격 가이드”** 표현 사용.
- **“AI가 문제를 만든다”** 대신 **“최신 기출 데이터를 학습한 AI 에이전트가 검수합니다”**로 서술.

## 2. 서비스 플로우

### 2.1 전체 사용자 여정 (고수준)

```
[비로그인] 홈 → 무료 진단 시작 → 모의고사 목록 → 퀴즈(20문항 제한) → 로그인 유도 / 결제 유도
[로그인 무료] 홈/마이페이지 → 모의고사 목록 → 1·2회차 풀기 → 3회차부터 결제 유도
[로그인 유료] 홈/마이페이지 → 모의고사 목록 → 전 회차 + Round 4+ 맞춤형 + 약점 공략 → 결과·대시보드
[결제] 회차 선택 → 결제 완료 → 이용권(pending) → 시험당일 12:00 KST 이후 활성화(active) → 열공모드 전체 기능
```

### 2.2 라우트와 진입점

| 경로 | 역할 | 비고 |
|------|------|------|
| `/` | Home | 비로그인 가능. 진단 시작·로그인·결제 유도. |
| `/login` | Login | 회원가입 포함. 게스트 이어풀기·결제 유도 후 복귀 처리. |
| `/mypage` | 나의 학습 대시보드 | 로그인 필수. D-Day, 성적 추이, 약점, 예측 합격률, 모의고사 진입. |
| `/account-settings` | 계정 설정 | 이용권 표시·회차 변경·주문 취소. |
| `/exam-list` | 모의고사 회차 목록 | certId 기준. 회차별 잠금(게스트/무료/유료). |
| `/quiz` | 퀴즈 진행 | 회차별 문항 로드, 제출 시 채점·통계·Elo 반영. |
| `/result` | 결과 | 과목별 점수, 합격 여부, 예측 합격률, 오답 해설(유료 시 AI 해설). |
| `/checkout` | 결제 | certId 기준, 회차 선택(또는 1개면 자동) → 결제 완료 후 마이페이지. |

### 2.3 시나리오별 상세 플로우

#### A. 비로그인(게스트) → 모의고사 → 로그인/결제

1. **홈**  
   “무료로 내 실력 진단받기” 또는 자격증 카드 클릭 → 모의고사 목록(`/exam-list`)으로 이동.
2. **모의고사 목록**  
   게스트는 무료 회차만 노출. 회차 선택 → 퀴즈(`/quiz`) 진입.
3. **퀴즈**  
   게스트 20문항 제한. 20문항 제출 시 “결과 저장 후 로그인하면 이어서 풀 수 있어요” 등 안내 → 로그인(`/login`) 이동.  
   또는 유료 회차/기능 클릭 시 결제 유도 → 로그인 후 `/checkout`.
4. **로그인/회원가입**  
   성공 시:  
   - 퀴즈 이어하기 대기 중이면 → 퀴즈로 복귀(이어풀기).  
   - 결제 유도로 왔으면 → 결제(`/checkout`) 이동.  
   - 그 외 → 마이페이지 등 안내.
5. **결제**  
   회차 선택(2개 옵션 시 모달, 1개 시 자동) → 결제 완료 처리 → 이용권(pending) 저장 → 마이페이지 이동.

#### B. 로그인 무료회원

- **1·2회차**: 풀기 가능.  
- **3회차 이상**: 잠금, 클릭 시 “열공모드로 합격권에 진입하세요” 등 안내 후 `/checkout` 이동.  
- **약점 공략·AI 해설**: 비프리미엄 시 마스킹 또는 결제 유도.

#### C. 로그인 유료회원(열공모드)

- **이용권(Pass)**  
  자격증별 1개. `round`, `examDate`, `status`(pending/active/cancelled).  
  **시험당일 12:00 KST 이후** 앱 진입 시 pending → active 전환, “열공모드 활성화” 팝업.
- **접근 가능 기능**  
  Round 1~3 고정 회차, Round 4+ 맞춤형(20문항), Round 5 약점 공략(80문항), AI 해설(aiExplanation), 대시보드 전체(약점 카드, 예측 합격률, 레이더·과목 밸런스).

#### D. 이용권 활성화(pending → active)

- **시점**: 해당 회차의 **시험당일 12:00 KST 이후** 앱 진입 시.  
- **처리**: 서버 시간(WorldTimeAPI 등) 또는 로컬 12:00 KST 기준으로 판단 후 Firestore `passes.{certCode}.status = 'active'` 갱신.  
- **주문 취소**: 해당 회차 **시험당일 12:00 이전**에만 가능.

### 2.4 데이터 관점 플로우 (간단)

- **인증** → Firebase Auth + Firestore `users/{uid}` (subscriptions, paidCertIds, passesByCert 등).
- **문제 선정**  
  - Round 1~3: 고정 시험 장부.  
  - **Round 4+(20문항)**: 맞춤 큐레이션 한 가지 — 맞춘 문제 제외, Zone A(틀린 문제) 우선 → Zone B(안 푼 문제)로 부족분, 과목별 비율 배분. UI에서만 D-Day·정답률에 따라 "실전 대비" vs "약점 강화" 추천 문구 표시.  
  - **Round 5(80문항)**: 실전 대비형(80% 트렌드·신규, 20% 오답) 또는 약점 강화형(헷갈림 24 + 오답 24 + 취약 태그 32). mode 없으면 D-Day 7일 기준으로 자동 비율 조절.
- **채점** → 제출 시 과목별 점수·합격 판정·exam_results 저장 → **정밀진단 데이터** 갱신: stats(3차원 통계·proficiency)·Elo·confused_qids.
- **대시보드(정밀진단)** → `users/{uid}/stats/{certCode}`·exam_results 기반 D-Day, 성적 추이, **약점 카드**(hierarchy/tag 숙련도), 예측 합격률, 레이더·과목 밸런스. 약점 우선순위는 `calculatePriority`(숙련도·망각·오개념)로 5회차 큐레이션에 반영.

## 3. 비즈니스 모델

### 3.1 수익화 구조

| 구분 | 내용 |
|------|------|
| **상품** | 자격증별 **이용권(Pass)** — “제 N회 시험 대비” 회차 단위. |
| **가격/정책** | (실제 가격은 사업계획서에서 정의) 회차 선택 후 결제, 시험당일 12:00 KST 기준 활성화. |
| **주문 취소** | 시험당일 12:00 이전만 가능. |
| **만료 후** | 재수강(예: 50% 할인) 유도. 오답 데이터 보관 안내로 재전환 강화. |

### 3.2 사용자 등급별 제공 가치

| 등급 | 정의 | 제공 내용 |
|------|------|-----------|
| **게스트** | 비로그인 | Round 1만 노출, 20문항 제한. 제출 후 로그인 유도·이어풀기(sessionStorage 복구). |
| **무료회원** | 로그인, 해당 자격증 미결제 | Round 1·2 회차 풀기. 3회차부터 결제 유도. AI 해설(aiExplanation) 마스킹. |
| **유료회원(열공모드)** | 이용권 active 또는 레거시 paidCertIds | 전 회차 + Round 4+ 맞춤형 + Round 5 약점 공략. AI 해설·대시보드 전체 오픈. |
| **만료** | 과거 유료, 현재 expired | 오답노트 열람 등 제한적 기능. 재수강 CTA. |

### 3.3 전환 유도 포인트

- **게스트**: 20문항 후 로그인 → 이어풀기·결과 저장 동기 부여.  
- **무료회원**: 2회차까지 체험 후 3회차·약점·AI 해설에서 “열공모드로 합격권에 진입하세요” 등 결제 유도.  
- **결제 완료**: pending → active 시점에 “열공모드 활성화” 팝업으로 인지도 상승.  
- **시험 후(D+1)**: 합격/불합격 설문 → 불합격 시 50% 할인 재수강 유도.

## 4. 사업계획 관련 정리

### 4.1 타깃 고객

- **1차**: 데이터·IT 자격증 수험생 (빅데이터분석기사, SQLD, ADsP 등).  
- **2차**: 자격증 로드맵(ADsP → SQLD → 빅데이터분석기사)을 따라 단계별로 도전하는 사용자 → LTV 확대.

### 4.2 가치 제안

- **데이터 기반 합격 가이드**: 기출 풀 기반 모의고사 + **정밀진단**(3차원 통계·Elo 숙련도) + 예측 합격률.  
- **맞춤형 회차**: Round 4+에서 맞춘 문제 제외, Zone A 복습 우선·Zone B 도전, 과목 비율 배분 20문항. D-Day·평균 정답률에 따라 실전 대비 vs 약점 강화 추천 문구만 표시(4회 문항 구성은 동일).  
- **정밀진단(학습자 진단)**: 제출 시마다 hierarchy·problem_type·subject·tag 별 통계·Elo 갱신. 대시보드에서 성적 추이·레이더·약점 카드로 진단, 4회차 추천·5회차 큐레이션에 반영.
- **약점 공략**: Round 5 실전형(8:2) / 약점형(헷갈림·오답·취약 태그 3:3:4), 약점 다시풀기로 confused_qids·proficiency 기반 취약 영역 집중.  
- **신뢰 메시지**: “최신 기출 데이터를 학습한 AI 에이전트가 검수” — 데이터 근거 명시.

### 4.3 경쟁 차별화(예시)

- **MAV(Multi-Agent Verification)** 개념: 지식·전략·경험 에이전트 협업 검수 (메인 화면 명세 기준).  
- **회차·이용권 모델**: 시험당일 12:00 KST 컷오프, pending/active 명확화로 일정 기반 운영.  
- **게스트 이어풀기**: 로그인 전 상태를 sessionStorage로 복구해 전환 장벽 완화.

### 4.4 성장 경로(로드맵)

- **자격증 로드맵**: ADsP(기초) → SQLD(중급) → 빅데이터분석기사(심화).  
- **기능 로드맵**: (실제 계획은 별도 문서에서) 예: 추가 자격증, PG 연동, AI 해설 고도화, 리포트 다운로드 등.

### 4.5 리스크·규정 대응

- **과장 광고 방지**: “100% 합격 보장” 금지, “합격 확률 최적화·데이터 기반 가이드”로 표현.  
- **개인정보**: 게스트 이어풀기는 sessionStorage, 탭 종료 시 초기화로 최소 데이터 유지.

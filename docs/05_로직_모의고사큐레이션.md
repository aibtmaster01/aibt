# 로직: 모의고사 큐레이션

> Round 1~3 고정형, Round 4+ 맞춤형(적응형) 문항 선정 로직. **실제 코드 기준** 정리.  
> 관련 파일: `src/services/examService.ts`, `src/services/aiRoundCurationService.ts`, `src/pages/ExamList.tsx`.

---

## 1. 회차 구분 (현재 정책)

| 구분 | 회차 | 문항 수 | 구성 방식 | 데이터 소스 |
|------|------|---------|-----------|-------------|
| **고정형** | 1, 2, 3 | 80 (빅데이터) | 인덱스에서 해당 round 필터 후 과목·core_id 순 정렬, 앞에서 N개 q_id 선정 | Storage/Firestore `index.json`, `certifications/{cert}/question_pools/contents_1681/questions/{q_id}` |
| **맞춤형** | 4 이상 | 80 | 과목별 20문항, 1→2→3→4 순서. 다양-적응 선발 (약점·트렌드·난이도·취약유형 가중) | 동일 풀 + `users/{uid}/stats/{certCode}`, `user_rounds` |

- **고정 4·5회차 제거됨.** 예전 "D-Day 3일·합격률 70% 언락" 조건 없음. 4회차부터 모두 **맞춤형 80문항**.
- **약점 공략 모의고사:** UI상 Round 6부터 "약점 공략 모의고사 1회", "2회" … 로 표시 (`ExamList`·`MyPage` 회차 라벨). 최대 회차는 `MAX_CURATION_ROUND`(예: 20)까지 동적 확장.

## 2. 고정형 (Round 1~3) 상세

- **진입점:** `examService.getQuestionsForRound(certId, round, user)`.
- **로직:**  
  1. `user`가 있으면 `users/{uid}/user_rounds/{round}` 존재 여부 확인. 있으면 저장된 `questionIds`로 즉시 풀 로드.  
  2. 없으면 **Static** 분기: `round <= 3` → 인덱스 캐시(`getQuestionIndexFromCache` / `syncQuestionIndex`)에서 `metadata.round === round` 인 항목만 필터.  
  3. `pickStaticRoundIdsInOrder`: 과목 번호 오름차순, 같은 과목 내 `core_id` 오름차순 정렬 후 앞에서 `needCount`개(기본 80) `q_id` 추출.  
  4. `fetchQuestionsFromPools(certCode, qIds)`로 Firestore에서 본문 로드.  
  5. 로그인 유저면 `user_rounds/{round}`에 `questionIds` 저장(트랜잭션, 기존 문서 있으면 덮어쓰지 않음) → **재응시 시 동일 문항 유지**.

- **문항 수:** `certification_info.subjects` 합계 또는 `STATIC_QUESTIONS_PER_ROUND[round]` (1~3 각 80).

## 3. 맞춤형 (Round 6+ / 약점 공략) 상세 — aiRoundCurationService

- **진입점:** 맞춤형 회차 시 `examService.getQuestionsForRound` → `aiRoundCurationService.fetchAdaptiveQuestions(uid, certId, user, round)` → **과목별 20문항×4과목(80문항)** 만 사용 (`selectQuestionIdsBy3ZonesPerSubject`).  
  - **레거시:** 전체 80문항 일괄 선발(`selectQuestionIdsBy3Zones`)은 제거됨.

- **상수 (aiRoundCurationService.ts):**
  - `QUESTIONS_PER_SUBJECT = 20`, 과목당 20문항 고정.
  - `COVERAGE_PER_SUBJECT = 12` — 1구역(커버리지) 슬롯 수.
  - `WEAKNESS_PER_SUBJECT = 8` — 2구역(약점 집중) 슬롯 수.
  - `MAX_PER_CORE_ID = 3`, `MAX_PER_SUB_CORE_ID = 2` — 동일 core_id/sub_core_id 최대 출제 수.
  - `EXCLUSION_WINDOW = 3`, `REUSE_FIXED_FROM_ADAPTIVE_N = 4` — 슬라이딩 윈도우·1~3회차 재사용 시점.
  - calcScore 가중치: `W_WEAKNESS`, `W_TREND`, `W_DIFF`, `W_WEAKTYPE`, `SCARCE_THRESHOLD`/`SCARCE_PENALTY`.

- **제외 집합:**  
  - `collectExcludedQIds(uid, currentRound)`: 직전 `EXCLUSION_WINDOW` 맞춤형 회차의 `user_rounds`에서 q_id 수집.  
  - 맞춤형 N번째가 `REUSE_FIXED_FROM_ADAPTIVE_N` 미만이면 1~3회차 `user_rounds`의 q_id도 선발 풀에서 제외.

- **풀 구성:**  
  - 인덱스: Storage `index.json` 또는 Firestore `certifications/{cert}/public/index` 캐시.  
  - round 99 항목 기본 사용. `adaptiveN >= REUSE_FIXED_FROM_ADAPTIVE_N`이면 round 1,2,3 항목도 풀에 포함(제외 q_id 제외).

- **과목별 선발 (`selectForSubject`):**  
  1. **1구역(커버리지):** core_id별 대표 문항 1개(calcScore 최고). core_id를 해당 sub_core_id 평균 proficiency 오름차순 정렬 후 상위 `COVERAGE_PER_SUBJECT`개 core_id의 대표만 선발.  
  2. **2구역(약점 집중):** 1구역 미선발 문항을 calcScore 내림차순으로, `MAX_PER_CORE_ID`/`MAX_PER_SUB_CORE_ID` 제약 하에 `WEAKNESS_PER_SUBJECT`개 선발. 부족 시 제약 완화(먼저 sub_core 제한 해제, 이후 core 제한 해제).  
  - **점수(calcScore):** WeaknessBonus + TrendScore + DifficultyFit + WeakTypeBonus + 희소 sub_core_id 페널티 + 랜덤 노이즈.

- **UserRound 저장:** 맞춤형 80문항 생성 후 `users/{uid}/user_rounds/{round}` 에 `questionIds` 저장(기존 문서 없을 때만). 재진입 시 동일 세트 유지.

## 4. UserRound (유저별 회차 고정)

| Firestore 경로 | 필드 | 설명 |
|----------------|------|------|
| `users/{uid}/user_rounds/{roundNum}` | `roundNum`, `sourceRounds`, `questionIds`, `createdAt` | 해당 회차에서 풀었던/풀 예정인 문제 ID 목록 |

- **역할:** 한 번 선정된 문항 세트를 재응시 시에도 그대로 사용.  
- **저장 시점:** Static/맞춤형 모두, 문항 로드 후 트랜잭션으로 "기존 문서 없을 때만" set.

## 6. 인덱스·풀 구조 (BIGDATA 기준)

- **문제 풀:** `certifications/BIGDATA/question_pools/contents_1681/questions/{q_id}`.  
- **인덱스:** Firebase Storage `assets/BIGDATA/index.json` 또는 Firestore `certifications/BIGDATA/public/index`.  
- 인덱스 항목: `q_id`, `metadata`(subject, round, core_id, sub_core_id, problem_type 등), `stats`(difficulty, trend 등).  
- **sync:** `localCacheDB.syncQuestionIndex(certCode)` → 캐시 후 `getQuestionIndexFromCache(certCode)`로 조회.

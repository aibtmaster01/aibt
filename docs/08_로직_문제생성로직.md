# 로직: 문제 생성 (AI-Gen Universal Framework)

> LLM 환각·포맷 붕괴 방지를 위한 Python 기반 문제 생성 파이프라인.  
> **프론트엔드 큐레이션과 별개** — 백엔드 `backend/Contents/` 구현.

---

## 개요

| 한계 | 대응 |
|------|------|
| 환각(Hallucination) | Pydantic CoT, Python Shuffle, 3-Tier Auditor |
| 포맷 붕괴 | JSON 스키마·Negative Constraints |
| 논리적 불일치 | 선지 타입 → 꼬리말 → 오답 기획 순차 통제 |

---

## 파이프라인 단계

| Step | 파일 | 역할 |
|------|------|------|
| 1 | `01_BluePrintExtracter.py` | TOC-Aware DNA 추출, Map-Reduce 압축 |
| 2 | `02_Generator.py` | Pydantic CoT, Python Shuffle, Adaptive Context |
| 3 | `03_3TierAuditor.py` | Logic/Style/Difficulty 3-Way Auditor |
| 4 | `SQLD_04_MetadataInjector.py` | Heuristic 메타데이터, 정답 인덱스 확정 |
| 5 | `SQLD_05_FinalAssembler.py` | R1~R4 퍼널 배정 |

## 빅데이터 콘텐츠·인덱스 업로드 (현재 운영 경로)

- **스크립트:** `backend/Contents/Bigdata/upload_contents_and_index.py`  
- **역할:**  
  - `Bigdata_contents_1681.json` 각 항목 → Firestore `certifications/BIGDATA/question_pools/contents_1681/questions/{q_id}`  
  - 같은 디렉터리의 인덱스 파일(`Bigdata_Index.json` 또는 `Index.json`) → Firebase Storage `assets/BIGDATA/index.json` 및 Firestore `certifications/BIGDATA/public/index`  
  - (선택) `Bigdata_Index_Rebalanced.json` 사용 시 Storage에 `index_rebalanced.json` 추가 업로드 가능 — 프론트 인덱스 동기화 시 우선 사용하도록 구현 가능.
- **실행:**  
  ```bash
  cd backend
  pip3 install -r requirements.txt   # firebase-admin 등
  export GOOGLE_APPLICATION_CREDENTIALS="/path/to/serviceAccountKey.json"
  python3 Contents/Bigdata/upload_contents_and_index.py
  ```
- **필수:** 서비스 계정 키에 Firestore + Storage 권한 필요.

## 프론트엔드와의 관계

- **프론트엔드는 업로드된 문제 풀(question_pools)과 인덱스(index.json)를 기반으로 큐레이션**함.
- 문제 **생성/업로드**는 백엔드 Python 스크립트, **선정·노출**은 examService·aiRoundCurationService·localCacheDB(인덱스 캐시).

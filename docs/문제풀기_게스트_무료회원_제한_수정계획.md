# 문제풀기 화면 – 게스트 20제 로그인 / 무료회원 2회 결제 유도

## 요구사항 요약
- **게스트**: 20문제 풀면 → 로그인 화면으로 이동
- **무료회원**: 2회차까지 풀 수 있고, 3회차부터는 결제 화면으로 이동

---

## 수정 대상 파일 및 계획

### 1. 게스트 20문제 → 로그인

| 파일 | 역할 | 수정 내용 |
|------|------|-----------|
| **`src/App.tsx`** | Quiz에 콜백 전달, 라우팅 | `Quiz`에 `onGuestLimitReached` prop 추가. 호출 시 `navigate('/login')` 실행. (기존에는 전달 안 함 → 3초 후 아무 동작 없음) |
| **`src/pages/Quiz.tsx`** | 20번에서 멈춤 + 모달 | 이미 구현됨: `GUEST_QUESTION_LIMIT = 20`, 20번 제출 시 모달 노출 후 `onGuestLimitReached` 호출. **수정 없음** (필요 시 모달 문구만 조정) |

**흐름**: 게스트가 20번 문제 제출 → 모달 "20번까지 결과 저장, 3초 후 로그인 화면으로 이동" → 3초 후 `onGuestLimitReached` → App에서 `/login` 이동.

---

### 2. 무료회원 2회까지 → 결제 화면

| 파일 | 역할 | 수정 내용 |
|------|------|-----------|
| **`src/pages/ExamList.tsx`** | 회차별 잠금/이동 | **이미 구현됨**: `getLockState`에서 `!isPremiumUser`일 때 `round >= 3` 잠금, `reason === 'free'` → 클릭 시 `onNavigate('/checkout')`. **변경**: 잠금 시 안내 문구를 "2회차까지 무료, 3회차부터 결제 후 이용"으로 명확히 수정. |
| **`src/pages/Result.tsx`** | (선택) 2회 완료 후 결제 유도 | 무료회원이 **2회차를 방금 완료**한 경우 "다음 회차(3회차)를 위해 결제하기" CTA 노출 가능. 적용 시: `App.tsx`에서 `handleQuizFinish`에 `roundId` 전달, Result에 `completedRoundId` 등 prop 추가 후 무료회원 && 2회차 완료 시 결제 버튼/문구 노출. |

**현재 흐름**: 무료회원이 1·2회차는 풀 수 있고, 3회차 카드를 누르면 이미 결제 페이지로 이동함. 문구만 정리하면 됨.

---

## 구현 순서 (권장)
1. **App.tsx**: Quiz에 `onGuestLimitReached={() => navigate('/login')}` 추가 → 게스트 20제 후 로그인 이동 완성.
2. **ExamList.tsx**: 무료회원 잠금 시 알림/툴팁 문구를 "2회차까지 무료입니다. 3회차부터는 결제 후 이용해 주세요" 등으로 수정.
3. (선택) **Result.tsx + App.tsx**: 2회차 완료 후 결제 유도 UI/이동 원하면, 위와 같이 roundId 전달 및 Result 분기 추가.

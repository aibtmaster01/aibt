# 전체 서비스 검토: 수정 파일 영향 및 가입·모의고사·결제 시나리오

## 1. 수정된 파일과 전체 서비스 충돌 검토

### 1.1 수정·삭제된 파일 목록 (이용권/시험일정 플랜 기준)

| 파일 | 변경 내용 | 충돌 여부 |
|------|-----------|-----------|
| `src/pages/Checkout.tsx` | 회차 선택 옵션 1개일 때 모달 없이 바로 결제 | **없음** – `onComplete(passInfo)` 시그니처 동일, 기존 호출부 그대로 동작 |
| `src/pages/AccountSettings.tsx` | 주문취소 조건을 `pass.examDate` 12:00 전으로 통일, `useEffect` import 추가 | **없음** – `cancelPass`와 UI 조건 일치 |
| `src/services/authService.ts` | `cancelPass`가 `pass.examDate` 기준 12:00 전만 취소 허용 | **없음** – Firestore `passes` 스키마와 호환 |
| `src/pages/GoalSetting.tsx` | **삭제** | **없음** – App.tsx에 `/goal-setting` 라우트 없음, 다른 파일에서 import 없음 |
| `src/components/MyPassDashboard.tsx` | `daysLeft == null` 문구 변경 (목표 시험일 설정 언급 제거) | **없음** – props(`daysLeft`, `onContinue` 등) 변경 없음 |
| `src/pages/MyPage.tsx` | `onRegisterGoal` prop 제거 | **없음** – App에서 해당 prop 전달하지 않음 |
| `src/App.tsx` | `pendingGoal` 상태 및 handleQuizFinish 내 해당 분기 제거 | **없음** – `setPendingGoal` 호출처 없어 미사용 코드 제거에 해당 |
| `src/components/AiLoadingOverlay.tsx` | `daysLeft == null` 시 문구 변경 | **없음** – 표시 문구만 변경, 로직/인터페이스 동일 |

### 1.2 연동 부위 검증

- **User / Firestore**
  - `User.passesByCert`, `PassInfo`, Firestore `users.passes`(certCode → PassInfo): 기존 구현 유지. `targetExamDateByCert`는 레거시·관리자용으로 유지되어 하위 호환됨.
- **D-Day / 시험일**
  - `getDaysLeft(certId)`, `getNearestExamDate(certId)`: `EXAM_ROUND_SCHEDULES` 기반 12:00 KST 컷오프 그대로 사용.  
  - `ExamList`: `user.passesByCert?.[certId]` 있으면 `getDaysLeftForDate(pass.examDate)`, 없으면 `getDaysLeft(certId)` 사용 → 이용권 있으면 이용권 시험일 기준 D-Day, 없으면 다음 회차 자동. **충돌 없음.**
- **AI 맞춤형 / 약점**
  - `examService.ts`: `targetExamDate = user.passesByCert?.[certId]?.examDate ?? user.targetExamDateByCert?.[certId]` → 이용권 우선, 없으면 기존 목표일. **충돌 없음.**
  - `aiRoundCurationService.ts`: `getTargetExamDate()`가 `passesByCert` 우선, `targetExamDateByCert` fallback. **충돌 없음.**
- **홈 / 진입**
  - `Home.tsx`: `hasHistory`에 `passesByCert`·`targetExamDateByCert` 둘 다 사용. `onStartDiagnostic(certId, result.dateId)` 호출, App의 `handleStartDiagnosticTest(certId, _dateId?)`는 `dateId` 미사용. **충돌 없음.**

**결론:** 현재 코드베이스 기준으로 수정·삭제된 파일은 전체 서비스와 **충돌 없음**. 레거시(`targetExamDateByCert`, `paidCertIds`만 있는 경우)는 유지·fallback으로 처리됨.

## 2. 전체 서비스 시스템 요약

### 2.1 역할별 구성

| 구분 | 설명 |
|------|------|
| **인증** | Firebase Auth + `AuthContext`. 로그인/회원가입 시 `authService.getSessionForCurrentAuth`로 Firestore `users/{uid}` 로드 → `User`(subscriptions, paidCertIds, passesByCert 등)로 매핑. |
| **이용권(Pass)** | 자격증별 1개. `round`, `examDate`, `status`(pending/active/cancelled), `roundChangeUsed`. 시험당일 12:00 KST 이후 진입 시 pending → active. 유료 기능 언락은 `status === 'active'` 기준. |
| **시험 일정** | `EXAM_ROUND_SCHEDULES`(회차·날짜), 12:00 KST 컷오프. D-Day·다음 회차는 `dateUtils.getDaysLeft` / `getNearestExamDate`로 자동 계산. |
| **모의고사** | `ExamList` → 회차 선택 → `Quiz` → 결과 저장(`gradingService.submitQuizResult`) + `ensureUserSubscription`. AI 맞춤형은 `examService`·`aiRoundCurationService`에서 `passesByCert.examDate` 또는 `targetExamDateByCert` 사용. |
| **결제(베타)** | PG 없이 시뮬레이션. `Checkout`에서 회차 선택(또는 1개면 자동) → `setPaymentComplete(uid, certId, passInfo)` → Firestore에 `memberships`, `paidCertIds`, `passes` 갱신. |

### 2.2 라우트와 진입점

- `/` : Home (비로그인 가능, 진단 시작·로그인·결제 유도)
- `/login` : Login (회원가입 포함)
- `/mypage` : MyPage (로그인 필요, 대시보드·이용권 표시)
- `/account-settings` : 계정 설정 (이용권·회차변경·주문취소)
- `/exam-list` : 모의고사 회차 목록 (cert 쿼리)
- `/quiz` : 퀴즈 진행
- `/result` : 결과
- `/checkout` : 결제 (certId 컨텍스트)

## 3. 가입 → 모의고사 진행 → 결제 시나리오 정리

### 3.1 비로그인(게스트) → 모의고사 → 로그인/결제

1. **홈(`/`)**  
   - “무료로 내 실력 진단받기” 또는 자격증 카드 클릭 → `onStartDiagnostic(certId, dateId)`  
   - `handleStartDiagnosticTest`가 `selectedCertId` 설정 후 `/exam-list`로 이동.

2. **모의고사 목록(`/exam-list`)**  
   - `certId` 기준 회차 목록 표시.  
   - 게스트는 무료 회차만 노출(`isPremiumUser === false`).  
   - 회차 선택 → `/quiz`로 이동 (필요 시 `roundId`, `certId` 유지).

3. **퀴즈(`/quiz`)**  
   - 20문항 제한 등 게스트 제한 적용.  
   - 제한 도달 시 `onGuestLimitReached` → `pendingGuestContinue` 설정 후 `/login` 이동.  
   - 또는 유료 회차/기능 클릭 시 `onRequestCheckout` → `pendingCheckoutCertId` 설정 후 `/login`.

4. **로그인(`/login`)**  
   - 로그인/회원가입 성공 시 `onAuthSuccess`:  
     - `pendingGuestContinue` 있으면 → 퀴즈로 복귀, 이어하기 모달.  
     - `pendingCheckoutCertId` 있으면 → `selectedCertId` 설정 후 `/checkout`.  
     - 그 외 신규 가입 시 → 마이페이지 유도 등.

5. **결제(`/checkout`)**  
   - `certId`(선택 자격증) 기준.  
   - `getRoundSelectionForCheckout(certId)`로 현재 시점 기준 (N회 / N+1회) 옵션 계산.  
   - **옵션 2개**: 회차 선택 모달 → 사용자 선택 후 `proceedPayment({ round, examDate })`.  
   - **옵션 1개**: 모달 없이 `proceedPayment(optionA)` 호출.  
   - 2초 후 `onComplete(passInfo)` → App의 `handleCheckoutComplete(passInfo)`.

6. **결제 완료 처리**  
   - `setPaymentComplete(uid, selectedCertId, passInfo)` 호출:  
     - Firestore `memberships`, `paidCertIds`, `passes`(round, examDate, status: 'pending') 갱신.  
   - `updateUser`로 로컬 `user`에 `passesByCert[selectedCertId]` 반영.  
   - 알림 후 `/mypage`로 이동.

### 3.2 로그인 회원 → 모의고사 → 결제

1. **홈 또는 마이페이지**  
   - “나의 학습 대시보드” 또는 “모의고사 목록” 진입 → `selectedCertId` 설정 후 `/exam-list`.

2. **모의고사 목록**  
   - `isPremiumUser === isPremiumUnlocked(user, selectedCertId)`  
     - 이용권 `status === 'active'` 또는 레거시 `paidCertIds` 포함 시 유료 회차 노출.  
   - D-Day는 `user.passesByCert?.[certId]` 있으면 해당 `examDate` 기준, 없으면 `getDaysLeft(certId)`(다음 회차 자동).

3. **퀴즈 → 결과**  
   - 결과 저장 후 “다음 회차” 등 클릭 시 유료 아니면 결제 유도 → `selectedCertId` 유지한 채 `/checkout`.

4. **결제**  
   - 위와 동일. 회차 선택(또는 1개면 자동) → `setPaymentComplete` → 마이페이지.

### 3.3 이용권 활성화(pending → active)

- **시점**: 해당 회차의 **이전 회차 시험당일 12:00 KST 이후** 앱 진입 시.  
- **처리**: App `useEffect`에서 `user.passesByCert`의 `status === 'pending'`인 항목을 검사 → `getUnlockDateForPass`, `isAfterNoonKST` 만족 시 `activatePassIfNeeded` 호출 → Firestore `passes.{certCode}.status = 'active'` 후 `getSessionForCurrentAuth`로 사용자 갱신, `setActivatedCertForModal`로 “열공모드 활성화” 팝업 표시.  
- 확인 시 마이페이지 해당 자격증 탭으로 이동(`mypageActiveCertId`).

### 3.4 계정 설정에서의 이용권

- **이용권 섹션**: 이메일과 쿠폰 사이.  
- **표시**: “{자격증명} 제{N}회 대비 \| 사용중 / N월 N일 이용 예정”.  
- **주문취소**: `status === 'pending'` 이고 **해당 회차 시험당일 11:59 이전**(`isBeforeNoonKST(pass.examDate)`)일 때만 노출.  
- **회차 변경**: D-7 이상, 맞춤형 미시행, `roundChangeUsed === false`일 때만 노출 (1회 한정).

## 4. 요약

- **수정 파일**은 기존 라우트·Firestore·레거시 필드와 **충돌 없이** 동작하도록 되어 있음.  
- **가입 → 모의고사 → 결제** 흐름은 게스트 제한 → 로그인/결제 유도 → Checkout 회차 선택(또는 1개 자동) → `setPaymentComplete` → 마이페이지로 일관되게 이어짐.  
- D-Day와 유료 언락은 **이용권(passesByCert)** 우선, 없으면 **다음 회차 자동** 및 **레거시 paidCertIds/targetExamDateByCert** fallback으로 정리됨.

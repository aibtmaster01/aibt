## 코드 동작 방식

### 오류 분류 (`getErrorCode`)

`src/utils/errorCodes.ts`에서 에러 메시지·코드를 정규식으로 매칭:

| 매칭 조건 | 반환 코드 |
|-----------|-----------|
| `code === 'permission-denied'` 또는 메시지에 "permission" | `ERR_FIREBASE_PERMISSION` |
| "문제를 불러올 수 없습니다" / "문제 로딩 실패" | `ERR_LOAD_QUESTIONS` |
| "접근이 제한" / "준비중" | `ERR_ACCESS_DENIED` |
| "시험 장부" / "등록되어 있지 않습니다" | `ERR_EXAM_CONFIG` |
| "해당 자격증" / "찾을 수 없습니다" | `ERR_CERT_NOT_FOUND` |
| "network" / "fetch" / "failed to fetch" | `ERR_NETWORK` |
| 그 외 | `ERR_UNKNOWN` |

### 표시 코드 변환

- `getDisplayErrorCode(internalCode)` → 유저 표시 코드 반환
- `DISPLAY_TO_INTERNAL` → 역방향 조회 (유저가 코드 알려줄 때)

## 사용처

| 파일 | 사용 방식 |
|------|-----------|
| `src/utils/errorCodes.ts` | 코드 정의·분류·변환 함수 |
| `src/services/examService.ts` | 문제 로드 실패 시 `getErrorCode` → `getDisplayErrorCode` 호출 |
| `src/components/ErrorView.tsx` | 에러 UI에 표시 코드 노출 |

---

## 코드 동기화 규칙

- 새 내부 코드 추가 시 `errorCodes.ts`의 `INTERNAL_TO_DISPLAY`와 이 문서를 함께 수정
- 유저가 오류 코드 제보 시 이 문서에서 **유저 표시 코드**로 검색하여 원인 파악

# 현재 이슈 및 상태 전달 (외부 개발자용)

## 1. Firestore 인덱스 (미해결 — 콘솔 작업 필요)

### 문제
- **문제풀기** 시도 시 아래 에러 발생 가능:
  ```
  The query requires a COLLECTION_GROUP_ASC index for collection questions and field q_id.
  ```

### 원인
- `src/services/examService.ts`에서 **1~4회차** 문제 로딩 시 `collectionGroup(db, 'questions')` + `where('q_id', 'in', chunk)` 쿼리를 사용함.
- 이 쿼리는 **단일 필드** collection group 인덱스(`questions` 컬렉션 그룹 + `q_id`)가 필요함.

### 해결 방법
- **단일 필드 인덱스**는 `firestore.indexes.json`으로 정의할 수 없고, Firebase 콘솔에서만 설정 가능.
- 에러 메시지에 포함된 **create_exemption** 링크를 브라우저로 열어 **인덱스 생성** 버튼 클릭.
- 예시 링크 (프로젝트 aibt-99bc6 기준):
  ```
  https://console.firebase.google.com/v1/r/project/aibt-99bc6/firestore/indexes?create_exemption=...
  ```
- 인덱스가 **Enabled** 될 때까지 1~2분 소요될 수 있음.

### 관련 파일
| 파일 | 역할 |
|------|------|
| `firestore.indexes.json` | 복합 인덱스만 정의. 단일 필드 인덱스는 여기 없음 (정책상 콘솔에서만 생성). |
| `src/services/examService.ts` | `fetchQuestionsFromPools()` → `collectionGroup('questions')`, `where('q_id', 'in', chunk)` 사용 (라인 554 근처). |

## 2. 추가로 발생할 수 있는 인덱스 에러 (5회차·폴백 경로)

### 상황
- **5회차(약점 공략)** 또는 **80문항 부족분 채우기** 시 `examService.ts`에서 아래와 같은 collection group 쿼리도 사용함:
  - `collectionGroup('questions')` + `where('cert_id', '==', ...)` + `where('random_id', '>=', ...)` + `orderBy('random_id')`
  - `collectionGroup('questions')` + `where('cert_id', '==', ...)` + `where('topic', '>=', ...)` + `where('topic', '<=', ...)`

### 대응
- 위 쿼리 실행 시 **복합 인덱스 필요** 에러가 나오면, 에러 메시지의 **인덱스 생성 링크**를 열어 생성하거나, `firestore.indexes.json`에 해당 복합 인덱스를 추가한 뒤 `npx firebase-tools deploy --only firestore:indexes` 로 배포.

## 3. Firestore 보안 규칙 (현재 적용 완료)

### 적용된 내용
- **collection group `questions`** 읽기: `match /{path=**}/questions/{doc} { allow read: if true; }`
- **users/{userId}/user_weakness_stats/{certCode}**: `read, create, update` 허용 (채점 결과·숙련도 저장용)

### 관련 파일
- `firestore.rules`

### 배포
- 규칙 수정 후 반영: `npx firebase-tools deploy --only firestore:rules`

## 4. 채점·결과 저장 (gradingService)

### 동작
- 퀴즈 종료 시 로그인 사용자면 `submitQuizResult(uid, certId, sessionHistory, questions)` 호출.
- `users/{uid}/user_weakness_stats/{certCode}`: 단원별 숙련도·오개념·last_attempted_at 갱신.
- `users/{uid}`: `elo_rating_by_cert` 필드 merge 업데이트.

### 관련 파일
| 파일 | 역할 |
|------|------|
| `src/services/gradingService.ts` | 채점 결과 제출, Elo·숙련도 업데이트 |
| `src/App.tsx` | `handleQuizFinish`에서 `submitQuizResult` 호출 |
| `src/pages/Quiz.tsx` | `onFinish(score, total, sessionHistory, questions)` 로 전달 |

# 정책: 회원 관리

> 가입·등급·접근 제어·탈퇴 등 회원 관련 정책. 실제 코드 기준.

---

## 1. 회원 등급 정의

`examService.ts` — `getUserGradeForCert(user, certId)` 기준.

| 등급 | 조건 | 접근 범위 |
|------|------|-----------|
| **Guest** | `user === null` 또는 해당 certId 미구독 | Round 1만 (20문항 제한) |
| **Free** | 로그인 + 구독 있음 + 미결제 | Round 1~2 |
| **Premium** | `paidCertIds`에 해당 certId 포함 (또는 admin) | Round 1~3 + Round 6~20 맞춤형 + 집중학습 |
| **Expired** | `expiredCertIds`에 포함 | 오답노트 열람만 |

### 등급 판별 로직 (코드)

```
user === null → Guest
user.isAdmin === true → Premium
certId not in user.subscriptions → Guest
certId in user.paidCertIds && certId in expiredCertIds → Expired
certId in user.paidCertIds → Premium
else → Free
```

## 2. 접근 제어

`examService.ts` — `checkExamAccess({ user, certId, round, ... })` 기준.

| 등급 | Round 1 | Round 2 | Round 3+ | 약점/집중학습 |
|------|---------|---------|----------|---------------|
| **Guest** | 20문항 제한 | 로그인 유도 | 로그인 유도 | 불가 |
| **Free** | 전체 | 전체 | 결제 유도 | 불가 |
| **Premium** | 전체 | 전체 | 전체 | 전체 |
| **Expired** | 불가 | 불가 | 불가 | 불가 |

### AI 해설 마스킹

`examService.ts` — `maskQuestionData(questions, grade)`:

| 등급 | `explanation` | `aiExplanation` | `wrongFeedback` |
|------|--------------|-----------------|-----------------|
| **Premium** | 노출 | 노출 | 노출 |
| **Guest/Free/Expired** | 노출 | **삭제** | UI 플레이스홀더 표시 |

- 플레이스홀더: `'가입 후 확인하기'`, `'열공모드 가입 후 오답인 이유 확인하기'`

## 3. 가입 플로우

### Firebase Auth + Firestore

| 단계 | 처리 | 파일 |
|------|------|------|
| 이메일/비밀번호 가입 | `createUserWithEmailAndPassword` → Firestore `users/{uid}` 생성 | `authService.ts` |
| Google 로그인 | `signInWithPopup(GoogleAuthProvider)` → Firestore 문서 확인/생성 | `authService.ts` |
| 유저 문서 생성 | `familyName`, `givenName`, `name`, `email`, `subscriptions`, `paidCertIds` 등 | `authService.ts` |
| 세션 복원 | `onAuthStateChanged` → `getSessionForCurrentAuth` | `AuthContext.tsx` |

### Firestore 유저 문서 구조

```
users/{uid}/
  email: string
  familyName: string
  givenName: string
  name: string
  isAdmin: boolean
  subscriptions: string[]      // 구독 중인 certId 목록
  paidCertIds: string[]        // 결제한 certId 목록
  expiredCertIds: string[]     // 만료된 certId 목록
  passesByCert: { [certCode]: PassInfo }
  elo_rating_by_cert: { [certId]: number }
```

## 4. 게스트 → 회원 전환

| 단계 | 동작 |
|------|------|
| 게스트 20문항 제출 | 결과 `sessionStorage`에 저장 → 로그인 유도 모달 |
| 로그인 성공 | `pendingGuestContinue` 플래그 → 퀴즈 복귀 (이어풀기) |
| 로그인 완료 토스트 | "로그인되었습니다" 2.5초 하단 표시 |

## 5. 이용권 활성화 (pending → active)

| 항목 | 내용 |
|------|------|
| **시점** | 시험당일 12:00 KST 이후 앱 진입 시 |
| **판단** | WorldTimeAPI(Asia/Seoul) 우선, 실패 시 로컬 시간 fallback |
| **처리** | Firestore `passes.{certCode}.status = 'active'` → "열공모드 활성화" 팝업 |
| **코드** | `App.tsx` `useEffect` → `activatePassIfNeeded` |

| 항목 | 현재 상태 |
|------|-----------|

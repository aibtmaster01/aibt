## ✅ 구현 체크리스트

### Phase 0: 사전 준비
- [ ] PG사 연동 (토스페이먼츠, 아임포트 등) 계약 및 키 확보
- [ ] Firestore `users/{uid}/coupons` 또는 `discounts` 스키마 설계
- [ ] 환급 처리 방식 결정 (계좌 입금 / 포인트 / 다음 결제 시 차감 등)

### Phase 1: 기본 결제
- [ ] 정가 39,000원 회차별 이용권 결제 플로우
- [ ] `Checkout.tsx` / `checkout` 라우트 연동
- [ ] 결제 완료 시 `passes`, `memberships` 갱신 (기존 authService 로직 확장)

### Phase 2: ① 첫 진입 (10,000원 할인)
- [ ] "해당 자격증 최초 결제" 판별 로직 (`users/{uid}/exam_results` 또는 `passes` 기반)
- [ ] 결제 페이지 진입 시 자동 할인 적용 또는 "첫 수강 1만 원 지원금" 배너
- [ ] 할인 코드 또는 PG사 쿠폰 연동 (선택)
- [ ] UI: "체감가 29,000원" 표시

### Phase 3: ② 합격 보상 (50% 환급)
- [ ] D+1 설문 "합격했어요" 응답 시 `hasAnsweredPostSurvey` + `passed` 플래그 저장
- [ ] 환급 대상 식별: 해당 회차 이용권 결제 이력 + 합격 확인
- [ ] 환급 금액 계산: `실결제액 * 0.5` (예: 29,000 → 14,500)
- [ ] 환급 처리: PG사 환불 API 또는 별도 입금 프로세스
- [ ] UI: "합격 축하 반값 장학금" 안내 문구

### Phase 4: ③ 재도전 (9,900원 패스)
- [ ] D+1 설문 "아쉬워요" 응답 시 `failed` 플래그 또는 재도전 대상 식별
- [ ] 재결제 시 9,900원 할인 자동 적용
- [ ] UI: "눈물의 9,900원 패스" 배너, "커피 두 잔 값으로..." 카피

### Phase 5: ④ 교차 학습 (10,000원 쿠폰)
- [ ] 자격증 A 합격/완료 시 자격증 B용 10,000원 쿠폰 발급
- [ ] `users/{uid}/coupons` 또는 `available_coupons` 저장
- [ ] 새 자격증 결제 시 쿠폰 선택 UI
- [ ] UI: "커리어 빌드업 응원금" 카피

### Phase 6: 마케팅·운영
- [ ] 각 정책별 카피라이팅 최종 반영
- [ ] 이메일/푸시 알림 (합격 시 환급 안내, 불합격 시 재도전 유도)
- [ ] 관리자 화면: 환급·쿠폰 발급 내역 조회
